Data specification::work
API endpoint: stelupa/scores-api/v1/works/

The work (being particular compositions or performances) API endpoint has data about works of composers and performers. Attributes include:
Attribute name Data type Description
collection integer Primary key of related collection. Null if not applicable.
order_in_collection integer Integer denoting order work appears in collection. Null if not applicable.
best_known_title varchar Title is generally very problematic to deal from a data standpoint. Some works will have well known titles that they are abbreviations, or idiomatic in some way, or specific to different languages. analytics is usually note included in this.
title_in_english varchar A verbatim translation to an english title.
title_in_original_language varchar Title used by the composer of performer.
identifying_code varchar Any specific identiying codes used to catalog the work (e.g a BWV number).
created_by integer Primary key of related musician.
publication_date_is_approximate boolean True if publication date is approximate.
composed_start_date_is_approximate boolean True if start date of composition is approximate.
composed_end_date_is_approximate boolean True if end date of composition is approximate.
publication_date date Date compositioni published (or released if recording).
composed_start_date date Date composition started.
composed_end_date date Date work on composition ended.
creation_location varchar Location where compostion was predominantly created.
creation_location_latitude float Decimal latitutde where composition was predominantly created.
creation_location_longitude float Decimal longitude where composition was predominantly created.
genre varchar General category of genre.
notes text Free text about work
tonality varchar Starting tonality of work. Null if not applicable.
instrumentation_summary varchar Summary of instrumental grouping used in work.
type_of_work varchar Category of work type (e.g. chamber, symphony, etc.).
tempo_remains_constant boolean True if tempo does not change throughout the duration of the work.
score_available boolean True if score avaialble via score explorer app.
verified boolean True if record has been verified.
