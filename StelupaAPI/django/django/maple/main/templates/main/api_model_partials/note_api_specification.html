Data specification::note
API endpoint: stelupa/scores-api/v1/notes/

The note API endpoint holds information about individual notes and rests that appear in socres. Attributes include:
Attribute name Data type Description
work integer Primary key of a related work which the note belongs to.
instrument integer Primary key of instrument playing note. Null if not applicable
has_instrumentation boolean If note has any associated instrumentation.
divisions integer Number of divisions that make up a quarter note. Note can change depending on the rhythmic complexity across the work as a whole.
time_signature_numerator integer Integer denoting the numerator of the time signature.
time_signature_denominator integer Integer denoting the denominator of the time signature.
current_measure integer Integer denoting the measure that the note occurs in.
midi_number integer Midi number denoting the pitch of the note (where middle C, or C5 is 60).
note_name varchar Human readable note name (i.e. A5, C#/Db6) denoting the note name and octave in which it occurs.
duration integer Duration of the note. Note that this should be used in conjunction with the divisions attribute. If divisions is listed as 25 (being the number of units tha make up a quarter note), a duration of 100 refers to 4 quarter notes, or a whole note.
is_rest boolean True the note is a rest value.
is_grace_note boolean If the note is a grace note. Grace notes have no duration in relation to the measure.
starting_location integer Starting loctation of the note in relation to the work.
starting_location_of_current_measure integer Starting location of the current measure the note is in.
quarter_beats_per_minute integer Indication of the tempo, in quarter beats per minute, when this note is played.
tie_status boolean Indication if the note is tied to the note preceeding it.
verified boolean True if record has been verified.
